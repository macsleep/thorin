# Schematics

The CPU is running at 8 MHz. The power on reset is generated by a NE555 timer to make the signal duration more precise. The MFP is connected to interrupt 5 of the CPU. The 68008 DIP CPU only has interrupt levels 2, 5 and 7 (non maskable) available. Normal DTACK generation usually contains a signal delay like a 74LS164 serial shift register. But the SRAM and EPROM used for Thorin are fast enough to keep up with the 8 MHz CPU (no delay needed). The BERR (Bus Error) signal is set to trigger after 162 clock cycles (the duration of a DIVS instruction). Power for the SBC is provided by a classic L78S05 linear voltage regulator. Power consumption is about 450 mA at 5 V (~2.5 Watt). A DC power supply in the range of 7.5-9 V is recommended. Any higher voltages will generate a lot of heat with the linear voltage regulator. Since the MFP UART does not provide RTS and CTS signals you can wire parallel port pins to provide those signals via software emulation. Timers C & D are hard wired for the UART to generate the different baud rates.
